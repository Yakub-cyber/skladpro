# Дорожная карта СкладПро

Анализ состояния (июль 2026), сравнение с рынком и приоритизированный план.

## Где мы сейчас

Технически это **не прототип, а ранний облачный SaaS**: мультитенант на Supabase
(auth, RLS, компании, приглашения сотрудников, автосинхронизация), с localStorage
как офлайн-кэшем/демо-режимом. ~11k строк, React 18 + Vite + Tailwind v4 + Zustand.

**Сильное:** движок проводки документов (проводка/отмена/откат), 7 типов складских
документов, мультисклад + ячейки, типы цен (розница/опт/в долг), долг контрагентов,
лояльность, RBAC (4 роли), доставка с реальной маршрутизацией (TSP + OSRM), разбор
накладной из текста (эвристики + опциональный LLM), онлайн-витрина.

**«ИИ» честно:** эвристики + опциональный DeepSeek-LLM. Работает, но пока не ML.

## Чего не хватает vs рынок (МойСклад, CloudShop, Контур.Маркет, 1С)

- **Инженерная зрелость:** ~~нет тестов/линтеров/типов~~ → базовый каркас добавлен
  (Vitest, ESLint, Prettier, CI-гейт). Осталось: TypeScript, покрытие компонентов.
- **Учётный минимум (parity):** партии + себестоимость (FIFO/средневзвеш.), резерв vs
  доступный остаток, печатные формы (ТОРГ-12, УПД, счёт, сч-ф) + НДС, стандартные
  ведомости с экспортом в Excel.
- **Закон РФ и интеграции:** 54-ФЗ/ОФД, реальный Честный знак (ГИС МТ), ЭДО (Диадок/СБИС),
  маркетплейсы (WB/Ozon/Яндекс — 81% e-commerce), банковские выписки.
- **Мобайл:** нет PWA (хотя сканер камерой и офлайн-стор готовы), нет React Native.

## Стратегия

Не догонять МойСклад по всему. Клин — **«ИИ-товароучётка для небольшого опта/дистрибуции
с сильной логистикой доставки»**: усиливать уникальное (разбор накладных + маршрутизация),
закрывая обязательный минимум учёта.
